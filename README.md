# Następny poziom debugowania kodu!

### Jak korzystać ze sprawdzarki v1.0 beta by Capybara1337?
## 1. Najpierw zainstaluj pliki: Kliknij przycisk Code, a nastepnie Download ZIP
![image](https://github.com/Capybara-1337/checker/assets/84173816/beb3ae90-7799-414d-b017-c6b6776b1278)
![image](https://github.com/Capybara-1337/checker/assets/84173816/111538c8-77af-4421-8662-8e306f8426ee)
## 2. Jeżeli chcesz zmienić ścieżkę do folderów w których powinny znajdować się pliki input, output oraz exe, włącz plik configure.exe i postępuj zgodnie z tym, co napisane masz na konsoli.
![image](https://github.com/Capybara-1337/checker/assets/84173816/dd5ecab3-cce5-416c-91d7-b138e80c6080)
### Jeżeli pasują ci podstawowe ustawienia, nie musisz ich zmieniać.
## Pamiętaj o poprawnym nazewnictwie plików:
### Jeżeli plik exe który testujesz nazywa się main.exe, pary plików input i output powinny nazywać się main1.in, main1.out, main2.in, main2.out itd. Każdy plik .in musi mieć swoją parę .out.
## 3. Po poprawnym włożeniu plików w odpowiednie foldery, włącz checker.exe oraz postępuj zgodnie z tym, co masz napisane w konsoli.
![image](https://github.com/Capybara-1337/checker/assets/84173816/07757409-254a-4c41-a5ce-8154c925eedf)
### Amount of tests - ilość testów które masz w odpowiednich folderach
### Name of the executable and tests - nazwa pliku .exe (powinna być taka sama jak .in oraz .out bez liczby określającej który to test)
## 4. W tym samym folderze w którym znajduje się checker.exe powinien znaleść się plik *nazwa pliku.exe*_results.txt. W nim masz informacje które testy zadziałały, a które nie.
![image](https://github.com/Capybara-1337/checker/assets/84173816/6be5fa1c-7645-4185-a246-758dd130ee62)
